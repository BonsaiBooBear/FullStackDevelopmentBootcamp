/*
  # Hoisting: Variables and function declarations are moved to the top of the execution context before executing
    - Access variables and functions before declaration
    - Initializations and assignments are NOT hoisted.
    - JavaScript will "hoist" declarations above the invocations because the invocations will only be referenced in the memory allocation stage.
    - JavaScript is an integrated language. No compiler needed.


    - Temporal Dead Zone (TDZ)
      - 'let' and 'const' keywords are hoisted, but they continue to remain in TDZ until they are assigned a value
      - JavaScript is aware of the variable/constant created using let/const, but it cannot be accessed until it has been declared/initialized
*/
