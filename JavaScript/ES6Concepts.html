/*
  # ES6 Concepts:
    - ES: ECMAScript (another name for JavaScript)
    - ES6: JavaScript 6th version (2015) - Last MAJOR update, became the new baseline
    - Several 'Syntactic sugar' (Minimalist syntaxes) were introduced in this version
    - ES6 is used extensively in JS libraries like React, Vue, Angular, etc.

    - Concepts/Syntaxes Introduced:
      1. let and const (Block scope) ***
        - let: Declare variables
        - const: Declare constants
      2. Template literals/strings (``)
        - It starts and ends with back ticks (`)
        - Supports multi-line strings and expressions
        - Syntax to use variables within template strings
          - `${variable}`
      3. Default parameters
        - Default values can be provided to the function parameters. If case arguments are not provided, the default values are used
        - Syntax: function abc(a = 0, b = 0) {}
      4. Arrow functions (() => {}) ***
        - Syntactic sugar for anonymous functions
        - Syntax:
          const abc = () => {
            console.log('Hi')
          }
          - Single line arrow function
            - Syntax:
              const abc = () => console.log('Hi')
            - Scenarios:
                - Executes a single expression (Skip {})
                - Single expression with a returns value (Skip {} and return keyword)
            - Additional note: Skip () if only ONE parameters exist
      5. Object property shorthand ({ prop1, prop2 })
        - Whenever the variable name and the object property name are the same, instead of 'key:value', only 'key' can be specified
      6. Destructuring ({ prop1 } = obj)
        - Create variables having the same name as property name and have the value = value of that property
          - Syntax:
            const { a, b } = obj1
            - Different variable name for properties
              const { a: test1, b: test2 } = obj1
      7. Spread Operator (...)
        - Used with arrays and objects
        - Syntax:
          [...arr] (Arrays)
          {...obj} (Objects)
        - Additional copy:
          - Shallow copy:
            - ONLY First order properties/elements are copied
            - Best for simple arrays and objects
          - Deep copy:
            - Complete copy of properties across ALL levels/orders
            - Best for complex arrays and objects

*/
