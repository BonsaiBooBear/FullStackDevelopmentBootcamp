  # JavaScript
    - Programming Language
    - Used in web development to make websites dynamic/interactive
    - Used to add logic in Client (FrontEnd-FE), Server (BackEnd-BE), Mobile Applications, etc.

  - Add JavaScript to HTML
    - script tag (Head/Body): If Body, put script toward the end so HTML/CSS on entire page can load.
      - Internal: <script>console.log('Hi')</script>
      - External: <script src="[link]"></script>

  - Data Types ***
    - Primitive
      - String (e.g. 'Jonathan', 'New York', 'A', etc.)
        - Category1: Name, City, etc. (alpha)
      - Number (e.g. 5, 37483499, 12.76, -102, etc.)
        - Category2: Age, totalMarks, etc. (numerical)
      - Boolean (e.g. true/false)
        - Category3: Are you a Minor?, Do you like Ice Cream? (Yes/No: 0 or 1)
      - null (Empty placeholder, set manually)
      - undefined (Default value)
    - Non Primitive
      - Objects

  - Variables & Constants
    - Store data
      - Variables: Changing over time (var)
      - Constant: Remains fixed over time (const)
    - Syntax:
      - var x = 10
      - const y = 20
    - Declare/Initialize (Either)
      - Declare: var x
      - Initialize: var x = 10
    - Assign: x = 20
    - Naming conventions for variable name:
      - Never start with number
      - Cannot contain spaces (single word - camelCase)
      - Cannot contain special characters other than $ or _
      - Cannot use reserved keywords (var, const, etc.)

- Display Output
  - Console: console.log('Hello World') **
  - alert(): Basic pop-up on the webpage
  - document.write(): Appends data to the HTML document (no longer is use)
  - document.getElementById('id').innerHTML: Add HTML inside an element with a specific ID
    
- Additional notes:
  - JS is loosely typed language
  - Comment
    - Single Line: //
    - Muli Line: /* */

# Future
  - Objects
  - var/let/const
